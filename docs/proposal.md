# Proposal: 智能拍照姿势与角度推荐系统

> 版本: v1.4 | 日期: 2026-05-26 | 状态: Phase 2 全部完成，进入 Phase 3

---

## 一、产品概述

### 1.1 一句话描述

一款基于 AI 视觉分析的 Android 拍照助手，根据用户所处的真实环境，通过 AR 实时叠加给出最佳的拍照姿势、站位和角度建议。

### 1.2 解决的核心问题

| 痛点 | 现状 | 我们的方案 |
|------|------|------------|
| 面对美景不知道怎么站、怎么摆 | 靠直觉乱拍，回去发现没一张能发 | AR 骨骼姿势线告诉你"就这样站" |
| 男朋友/路人拍照技术差 | 构图靠运气 | 实时构图参考线 + 机位引导 |
| 专业摄影知识门槛高 | 光比、构图、景深... 普通人不懂 | AI 替你懂，转译成"往左一步""脸转过来" |
| 看到好看的照片不知道怎么模仿 | 存图对比，自己拍还是不像 | "姿势克隆"一键提取骨骼，AR 引导复刻 |
| 拍完后才知道不好看 | 无法挽回 | 拍后即时分析，下次拍更好 |

### 1.3 目标用户

- **主用户群**: 18-35 岁女性，有社交分享需求，对照片质量有要求
- **次用户群**: 旅行爱好者、摄影新手、社交媒体内容创作者
- **延伸用户**: 专业摄影师（取景辅助）、情侣/闺蜜（合照指导）

### 2.2 核心用户流程

#### 主流程：自拍姿势引导

```
1. 用户到达拍照地点
2. 打开 APP，相机启动
3. 端侧自动分析场景（2秒内完成）
4. APP 推荐 5 个适合当前场景的姿势
5. 画面出现 AR 骨骼姿势线 + 站位指示区域
6. 用户走过去，实时看到自己的骨骼和目标骨骼的对比
7. 对齐度指示条变绿 → 满意 → 拍照
8. 照片保存，系统给出评分和改进建议
9. 如果评分低 → "再来一次" → 回到步骤 4
```

#### 次流程：摄影师模式

```
1. 切换到摄影师模式
2. 对准场景，APP 分析构图
3. 画面出现构图参考线 + 建议拍摄角度提示
4. APP 语音提示："试试蹲低一点，从下往上拍"
5. 提供 3 个候选机位（红色圆点标记地面位置）
6. 用户选择机位 → AR 导航走到最佳位置
7. 到达后提示最佳焦段和参数
8. 拍摄
```

### 2.3 关键交互细节

| 交互场景 | 用户操作 | 系统响应 |
|----------|----------|----------|
| 切换姿势 | 左右滑动底部姿势卡片 | AR 骨骼动画流畅切换到新姿势 |
| 姿势不满意 | 下拉刷新 / 点击"换一批" | 云端返回新一批 5 个姿势 |
| 锁定推荐 | 长按姿势卡片 | 固定当前推荐，后续只微调 |
| 切换到后置 | 点切换摄像头按钮 | 自动进入摄影师模式 |
| 看不到 AR 骨骼 | 后退几步 / 站远点 | 提示"请后退以使骨骼线可见" |
| 网络差 | — | 自动切换到本地推荐模式，UI 无感 |

## 三、MVP 分期计划

### Phase 1 — 姿势推荐内核（MVP，预计 3-4 个月）

**交付物：可用的 APK，在单一场景下跑通全链路。**

| 模块 | 范围 | 不包含 |
|------|------|--------|
| 场景识别 | 5 类场景（户外自然/街拍/室内/海滩/夜景） | 细粒度场景、POI 感知 |
| 姿势库 | 100 个基础单人姿势 | 多人姿势、姿势克隆 |
| AR 引导 | 骨骼线条叠加 + 站位指示 | 骨骼对齐度评分、动画过渡 |
| 推荐引擎 | 场景匹配 → Top 5 姿势 | 个性化推荐、用户画像学习 |
| 光线分析 | 方向检测 + "面向光源"提示 | 光质分析、逆光剪影建议 |
| 离线模式 | 本地 100 姿势 + 规则引擎 | 离线 LLM |
| 用户系统 | 基础注册 + 风格偏好问卷 | 动态画像、成长曲线 |
| UI | 核心拍照界面 | 姿势广场、地图、社交 |

### Phase 2 — 全场景 + 全人群（预计 3-4 个月）

**计划范围（目标）：**

- 场景扩展至 100+ 细粒度分类

- 个性化推荐上线（基于历史的动态画像）

- 光线完整分析（光质、色温、逆光策略）
- 相机参数双层指导（小白模式 + 进阶模式 + 姿势-参数联动）
- 服装道具场景推荐

## 四、非功能需求

### 4.1 性能要求

| 指标 | 目标 | 测量方式 |
|------|------|----------|
| 相机启动到首次推荐 | < 3 秒 | 端侧冷启动计时 |
| AR 骨骼渲染帧率 | ≥ 30 FPS | 开发工具帧率监控 |
| 端侧推理延迟（每帧） | < 100ms（所有模型总计） | 各模型推理时间加总 |
| 云端推荐响应 | < 2 秒（WiFi） | P95 延迟 |
| APK 体积（含模型） | < 150MB | 构建产物大小 |
| 运行时内存 | < 500MB | 内存监控 |

### 4.2 隐私与安全

- 人脸/人体检测、场景分类全部在端侧完成，**不上传原始图片到云端**
- 云端仅接收匿名化的特征向量、骨骼坐标（仅坐标值，无关身份）、光线参数
- 用户拍摄的照片仅存储在本地设备
- 姿势广场 UGC 上传需经 NSFW 内容审核
- 隐私政策需在首次启动时展示并获取同意
- 符合《中华人民共和国个人信息保护法》要求

### 4.3 兼容性

- Android 10（API 29）及以上
- ARCore 支持的设备（Google Play Services for AR）
- 适配主流分辨率（1080p / 1440p / 4K）
- 支持主流国产手机品牌（华为/小米/OPPO/vivo/荣耀）

## 五、竞品分析

| 产品 | 核心功能 | 我们的差异点 |
|------|----------|-------------|
| 小红书相机 | 滤镜+美颜 | 我们做的是**姿势指导**，不是后期 |
| 轻颜/无他 | 美颜+美妆 | 我们不做美颜，做**构图和姿态** |
| VSCO/Snapseed | 专业后期调色 | 我们是**拍前优化**，不是拍后 |
| 摄影师 AI 助手类 APP | 风光构图建议 | 我们覆盖**人像姿势**，更大众 |
| PoseCam/Pose 指南类 | 静态姿势图片浏览 | 我们**实时 AR 叠加**，沉浸式引导 |

**核心差异化**：目前市场上没有一款产品能做到 "打开相机 → AI 理解场景 → AR 实时显示姿势骨骼 → 用户对齐骨骼 → 精准拍照" 的完整闭环。

## 七、风险与对策

| 风险 | 影响 | 概率 | 对策 |
|------|------|------|------|
| ARCore 兼容性差（国产手机） | 核心功能不可用 | 中 | 备选方案：2D 叠层方案（非 AR 模式），在无 ARCore 设备上降级为透明骨骼图片叠加 |
| 端侧模型精度不够 | 推荐质量差 | 中 | MVP 优先用云端推理保证质量，端侧做 fallback |
| 国产大模型多模态能力不足 | 姿势推荐质量差 | 低 | 多模型兜底（Qwen-VL + GLM-4V 双路），加规则过滤 |
| 用户不接受 AR 骨骼叠加 | 留存低 | 低 | 同时提供简化视图（仅文字提示+站位圈），用户可选 |
| 云端推理成本过高 | 无法盈利 | 中 | 端侧离线优化、姿势库缓存、批处理推理 |
| 隐私合规风险 | 下架 | 中 | 端侧优先架构、不传输原始图片、合规审查前置 |

## 九、附录：术语表

| 术语 | 定义 |
|------|------|
| 姿势骨骼 | 人体关键点（肩/肘/腕/髋/膝/踝/头）连接成的线条图，用于 AR 叠加 |
| 骨骼对齐度 | 用户当前姿势骨骼与推荐目标骨骼之间的空间相似度，用于实时评分 |
| 机位 | 拍摄者站立的最佳位置，由空间分析和构图规则计算得出 |
| 姿势克隆 | 从任意照片中提取人体骨骼，生成可复用的姿势数据 |
| 黄金时刻 | 日出后 1 小时和日落前 1 小时，光线最柔和适合拍照 |
| 特征向量 | 场景分析产出的一组数值（非图片），用于云端推理而不泄露隐私 |
